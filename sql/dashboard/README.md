# Retail Analytics (KPI metrics)

### 1. Total number of orders count

**Definition** : Number of distinct sales invoices generated.

**Formula**
```

COUNT(DISTINCT fact_sales.invoice_id)


```

**Notes**
- One invoice may contain multiple items

---

### 2. Total Units Sold

**Definition** : Total number of product units sold.

Formula
```

SUM(fact_sales.quantity)


```
---

### 3. Total Revenue
**Definition** : Total revenue generated from completed sales transactions.

**Formula**
```

SUM(fact_sales.line_total)


```
**Notes**
- Excludes returns
- Net revenue shown separately

---

### 4. Total Refund Amount

**Definition** : Total monetary value refunded to customers for returned items.

**Formula**
```

SUM(fact_returns.refund_amount)


```

**Notes**
- Based on recorded return transactions
- Refunds reflect final paid amounts

---

### 5. Return Rate

**Definition** : Percentage of sold units that were returned.

**Formula**
```

(SUM(quantity_returned) / SUM(quantity_sold)) × 100


```
**Notes**
- Quantity-based (not invoice-based)
- Sales and returns are aggregated independently

---

### 6. Monthly Revenue Summary

**Definition** : Revenue trend aggregated by calendar period.

**Notes**
- Based on transaction date
- Uses calendar dimension for consistency

---

### 7. Total Units Sold
**Definition** : Number of product units sold over time.

---

### 8. Revenue by Store

**Definition** : Revenue contribution by individual store locations.

---

### 9. Top Customers by Revenue

**Definition** : Customers ranked by total revenue contribution.

**Notes**
- Based on sales only
- Does not subtract refunds

---

### 10. Revenue by Category

**Definition** : Revenue generated by each product category.

---

### 11. Revenue by Sub-Category

**Definition** : Revenue generated by each product sub-category.

---

### 12. Top Products by Revenue

**Definition** : Products ranked by total sales revenue.

---

### 13. Monthly Refund Amount

**Definition** : Trend of refunded amounts monthly.

---

### 14. Return Rate by Category

**Definition** : Percentage of units returned within each product category.

**Formula**
```

(Category units returned / Category units sold) × 100


```

**Notes**
- Quantity-based
- Computed using independent aggregates

---

### 15. Top Returned Products

**Definition** : Products with the highest number of returned units.

---

### 16. Revenue by Discount 

**Definition** : Revenue generated from transactions where discounts were applied.

**Notes**
- Exact metric
- Discount sourced from sales data

---

### Data Notes:
- Sales and returns are modeled as separate event tables
- Return rates are quantity-based
- Discount-related return rates are approximate and used for trend analysis only
